## Product Requirements Document (PRD) untuk Sistem Absensi Mahasiswa

### 1. Tujuan dan Manfaat

Tujuan dari Sistem Absensi Mahasiswa adalah untuk memudahkan proses absensi mahasiswa, meningkatkan efisiensi, dan meminimalkan kesalahan input. Manfaat dari sistem ini adalah:

* Meningkatkan akurasi data absensi mahasiswa
* Memudahkan dosen untuk menginput absensi mahasiswa
* Memungkinkan perubahan data absensi jika terjadi kesalahan input
* Menampilkan rekap absensi mahasiswa

### 2. Spesifikasi Teknis

* Sistem Absensi Mahasiswa akan dibangun menggunakan teknologi web dan database.
* Sistem akan memiliki antarmuka pengguna yang user-friendly dan mudah digunakan.
* Sistem akan memiliki fitur validasi bentrok jadwal untuk mendeteksi jadwal yang High Jadwal bentrok.
* Sistem akan memiliki fitur input absensi dosen untuk menginput absensi mahasiswa setiap pertemuan.
* Sistem akan memiliki fitur edit absensi untuk memungkinkan perubahan data absensi jika terjadi kesalahan input.
* Sistem akan memiliki fitur rekap absensi untuk menampilkan rekap absensi mahasiswa.

### 3. Pengguna

* Pengguna utama sistem adalah dosen dan mahasiswa.
* Dosen akan menggunakan sistem untuk menginput absensi mahasiswa setiap pertemuan.
* Mahasiswa akan menggunakan sistem untuk melihat rekap absensi mereka.

### 4. Tim Pengembang

* Tim pengembang akan terdiri dari 3 orang: 1 orang untuk desain antarmuka pengguna, 1 orang untuk pengembangan backend, dan 1 orang untuk pengembangan frontend.
* Tim pengembang akan bekerja sama untuk mengembangkan sistem Absensi Mahasiswa.

### 5. Metrik dan Target

* Metrik:
 + Akurasi data absensi mahasiswa: 95%
 + Waktu penginputan absensi dosen: 2 menit
 + Waktu perubahan data absensi: 1 menit
 + Waktu menampilkan rekap absensi: 1 menit
* Target:
 + Mengembangkan sistem Absensi Mahasiswa dalam waktu 3 bulan
 + Melakukan testing dan validasi sistem dalam waktu 1 bulan
 + Melakukan pelatihan pengguna dalam waktu 1 bulan

### 6. Rencana Implementasi

* Rencana implementasi sistem Absensi Mahasiswa adalah sebagai berikut:
 + Bulan 1: Mengembangkan desain antarmuka pengguna dan backend
 + Bulan 2: Mengembangkan frontend dan melakukan testing dan validasi
 + Bulan 3: Melakukan pelatihan pengguna dan melakukan implementasi sistem

Dengan demikian, Sistem Absensi Mahasiswa dapat memudahkan proses absensi mahasiswa, meningkatkan efisiensi, dan meminimalkan kesalahan input.
